Operation: Rewind

플레이어의 선택에 따라 달라지는 NPC 패턴을 분석해 완벽한 루트를 찾는 루프형 잠입 액션

Unity 6.0 C# PC 팀 4인 (프로그래머) 기간: 주말 포함 35일

게임 개요

  • 루프 구조: 실패하면 시간은 초기화되지만, 플레이어는 NPC 패턴을 숙지하며 최적 루트를 탐색합니다.
  • 핵심 재미: 선택→반응→학습의 반복. 같은 상황도 선택에 따라 NPC의 상태/행동이 달라집니다.

담당 파트

이벤트 시스템

  • 모든 이벤트를 열거형으로 관리하여 오타 방지·가독성 향상
  • 구독/구독해제/트리거 3개 함수로 제어하는 중앙 집중식 설계
  • NPC·오브젝트·UI가 직접 연결되지 않고 이벤트로만 통신(느슨한 결합)

NPC FSM

  • NPCFSM 베이스에Idle/Look/Walk/Run/Talk/Dead 상태 정의
  • 공통 상태/전이를 공유하고, 은행원/경찰/시민 등 역할별 오버라이드로 행동 확장
  • 죽음 애니메이션 종료 후 Ragdoll 활성화(물리 전환) 처리

리플레이 & 스크린샷

  • 주요 순간을 화면 캡처 → **리스트/큐**에 시간순 저장
  • 플레이어 사망 시 저장된 이미지를 순서대로 출력하며 원인 추적을 돕는 리플레이
  • 영상 전체 녹화 없이 필요한 순간만 기록해 성능 부담 최소화

데이터베이스

  • Firebase Realtime Database로 **로그인/진행도/NPC 대사** 관리
  • 클라우드 데이터가 UI·대화에 즉시 반영되도록 구조화

UI / UX

  • 로그인/회원가입, NPC 상태표시, 사운드 매니저 등 **필수 HUD** 구성
  • 학습을 돕는 피드백: 트리거 결과와 NPC 반응을 즉시 확인 가능

코드 스냅샷

핵심 베이스 FSM 구조(요약)

State { Idle, Look, Walk, Run, Talk, Dead }
ChangeState(newState) -> 애니메이터 트리거 초기화/설정
DeadState() -> Dead 애니 후 Ragdoll 전환
TalkState() -> E 대화 시작 / F 강제 처치, 관련 시스템(Outline/Agent/Chat) 제어
